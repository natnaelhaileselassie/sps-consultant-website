# Possible Future Requirements
Below describes possible future functional and nonfunctional requirements which were not included in the current-requirements.md file. These lists were separated as the following may change over time. This keeps the process agile and flexible.

# Functional
The system shall…
- allow users to create an account and log in safely
- allow users to securely book and pay for services
- allow users to register for conferences which either allow them to pay or gives them instructions on how to pay 
- have a discussion form which allows users to interact
- possibly have capacity to create groups

# Nonfunctional
The system shall…
- maintain a level of security to protect sensitive user information
- use secure systems to handle financial transactions
